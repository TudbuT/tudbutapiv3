TudbuT API, the third.



The TudbuT API has existed for quite a while now, and its current version is... less than ideal. It's hard to maintain (giant switch statements are what routes traffic), the database is old and could use a revamp, the UserRecord model bugs out when the user uses multiple things at once.

Let's fix that.

The third iteration of my API is running on Tryumph, a robust HTTP server written in Java, able to route traffic using an annotation-based event distrubution system, and with numerous parsers built right into the server.

Getting to the point

The API has been completely remade, now with support for multiple services, so other clients by myself and others can use the API aswell. It's also much cleaner code, meaning I'll be able to add features much more easily.

FAQ

  • Question: What about old data?
    Answer: Anything that can be migrated, will be.
  • Question: Will my playtime be reset?
    Answer: No.
  • Question: Will there be a download of the old database?
    Answer: Yes, partially.
  • Question: Is the new API going to use JSON?
    Answer: For each endpoint, it will have JSON formatting, and TCN (a more readable format) formatting.
  • Question: When will the source code be published and where?
    Answer: Probably GitHub, maybe GitLab, and as soon as I'm decently confident there're no vulnerabilities (ETA 10 days after launch of API).
  • Question: Can I use the API for my client?
    Answer: Yes, most likely. DM TudbuT#2624 on Discord or write to api@mail.tudbut.de and provide details, I will consider adding a service. If so, you will receive a key with which you can manage the service. Coding knowledge required!!!
