TudbuT API, the third.



The TudbuT API has existed for quite a while now, and its current version is... less than ideal. It's hard to maintain (giant switch statements are what routes traffic), the database is old and could use a revamp, the UserRecord model bugs out when the user uses multiple things at once.

Let's fix that.

The third iteration of my API is running on Tryumph, a robust HTTP server written in Java, able to route traffic using an annotation-based event distrubution system, and with numerous parsers built right into the server.

For example, it can detect:
Your browser's fingerprint:
right on the server side, and without any JavaScript required. It can also modify the HTML being sent on-the-fly, and inject mobile-compatibility code.

The API is also using an entirely new data model: A UserRecord points to multiple ServiceRecords which link to their ServiceData, which is data about what the service is, how many people use it, etc.
There will be a Discord service aswell, so users can link their Discord account. Because of this, I could make a Discord bot that tells people how much playtime they have, and other statistics.
